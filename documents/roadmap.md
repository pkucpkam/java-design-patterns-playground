# Design Patterns Learning Roadmap

This roadmap is designed to guide you through the world of Design Patterns, from the most fundamental concepts to more advanced applications. The suggested times are estimates for understanding the core concept and the code example. True mastery comes with practice.

---

## Level 1: Beginner (Focus: Core Concepts)

These are the most common and foundational patterns. Mastering them will give you the biggest return on investment in your daily coding.

1.  **Strategy**: Learn to define a family of algorithms, encapsulate each one, and make them interchangeable. *(Suggested Time: ~2 hours)*
2.  **Singleton**: Ensure a class has only one instance and provide a global point of access to it. *(Suggested Time: ~1.5 hours)*
3.  **Facade**: Provide a simplified interface to a complex system of classes. *(Suggested Time: ~1.5 hours)*
4.  **Observer**: Define a one-to-many dependency between objects so that when one object changes state, all its dependents are notified. *(Suggested Time: ~2 hours)*
5.  **Decorator**: Add new functionality to an object dynamically without altering its structure. *(Suggested Time: ~2.5 hours)*

---

## Level 2: Intermediate (Focus: Object Creation & Structure)

Once you're comfortable with the basics, you can move on to these patterns which deal with more complex object creation and structural challenges.

1.  **Factory Method**: Define an interface for creating an object, but let subclasses alter the type of objects that will be created. *(Suggested Time: ~2 hours)*
2.  **Abstract Factory**: Create families of related or dependent objects without specifying their concrete classes. (Learn this after Factory Method). *(Suggested Time: ~2.5 hours)*
3.  **Builder**: Separate the construction of a complex object from its representation. *(Suggested Time: ~2 hours)*
4.  **Adapter**: Allow objects with incompatible interfaces to collaborate. *(Suggested Time: ~1.5 hours)*
5.  **Proxy**: Provide a surrogate or placeholder for another object to control access to it. *(Suggested Time: ~2.5 hours)*
6.  **Template Method**: Define the skeleton of an algorithm in an operation, deferring some steps to subclasses. *(Suggested Time: ~2 hours)*

---

## Level 3: Advanced (Focus: Complex Control Flow & Behavior)

These patterns are more specialized and often solve complex problems related to how objects communicate and control flow.

1.  **Command**: Turn a request into a stand-alone object. This lets you parameterize clients with different requests, queue requests, and support undoable operations. *(Suggested Time: ~3 hours)*
2.  **Chain of Responsibility**: Pass a request along a chain of handlers. Upon receiving a request, each handler decides either to process it or to pass it to the next handler in the chain. *(Suggested Time: ~2.5 hours)*
3.  **State**: Allow an object to alter its behavior when its internal state changes. The object will appear to change its class. (Often compared with Strategy). *(Suggested Time: ~3 hours)*

---

## Putting It All Together

The final step is to review the **"Ứng dụng trong Spring Boot"** section of each pattern. This will help you see how these theoretical patterns are applied in a real-world, modern framework, cementing your understanding.

By following this roadmap, you'll build a strong foundation in Design Patterns and be able to apply them effectively in your own projects.
